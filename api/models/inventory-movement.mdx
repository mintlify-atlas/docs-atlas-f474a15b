## Overview

The `InventoryMovement` model represents a single inventory transaction, tracking stock additions (inbound) or removals (outbound) for a specific product. This model implements an event-sourcing approach where the current stock balance is calculated by aggregating all movements.

<Note>
  This domain model is located in the core layer (`core/src/main/java/com/fbaron/ims/inventory/model/`) and is part of the Inventory bounded context.
</Note>

## Model Structure

```java
package com.fbaron.ims.inventory.model;

import com.fbaron.ims.product.model.Product;
import lombok.AllArgsConstructor;
import lombok.Builder;
import lombok.Getter;
import lombok.NoArgsConstructor;
import java.time.LocalDateTime;
import java.util.UUID;

@Getter
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class InventoryMovement {

    private UUID id;
    private Product product;
    private Integer quantity;
    private MovementType type;
    private LocalDateTime createdAt;
    private String reason;

}
```

## Fields

<ParamField path="id" type="UUID">
  **Movement Identifier**
  
  Unique identifier for this specific inventory movement transaction. Each movement is treated as an immutable event in the system.
  
  - **Generated by:** Database/Repository layer
  - **Immutable:** Yes
  - **Example:** `7c9e6679-7425-40de-944b-e07fc1f90ae7`
</ParamField>

<ParamField path="product" type="Product">
  **Associated Product**
  
  Complete Product entity associated with this movement. This establishes the relationship between inventory movements and products.
  
  - **Required:** Yes
  - **Type:** Domain object reference
  - **Contains:** Full product details (id, name, price, description, category)
  - **Relationship:** Many-to-One (many movements can reference one product)
</ParamField>

<ParamField path="quantity" type="Integer">
  **Movement Quantity**
  
  The number of units involved in this movement. Always represents an absolute positive value, with direction indicated by the `type` field.
  
  - **Required:** Yes
  - **Constraints:** Must be positive (> 0)
  - **Example:** `100`, `25`, `1`
  - **Validation:** `@Positive` at DTO level, also validated in business logic
</ParamField>

<ParamField path="type" type="MovementType">
  **Movement Type**
  
  Indicates the direction of the movement. This enum determines whether stock is being added or removed.
  
  - **Required:** Yes
  - **Values:**
    - `INBOUND` - Stock addition (increases balance)
    - `OUTBOUND` - Stock removal (decreases balance)
  - **Set by:** Business logic based on the endpoint called
</ParamField>

<ParamField path="createdAt" type="LocalDateTime">
  **Creation Timestamp**
  
  The exact date and time when this movement was recorded in the system. Used for auditing and historical tracking.
  
  - **Required:** Yes
  - **Generated by:** Business logic on movement creation
  - **Format:** ISO 8601 (e.g., `2026-03-11T10:30:00`)
  - **Immutable:** Yes (movements are never modified)
  - **Timezone:** System default (typically UTC in production)
</ParamField>

<ParamField path="reason" type="String">
  **Movement Reason**
  
  Human-readable description explaining why this movement occurred. Essential for auditing and traceability.
  
  - **Required:** Yes
  - **Constraints:** Cannot be blank
  - **Examples:**
    - `"Initial stock from supplier"`
    - `"Sale order #12345"`
    - `"Damaged goods - warehouse inspection"`
    - `"Return from customer"`
  - **Validation:** `@NotBlank` at DTO level
</ParamField>

## MovementType Enum

```java
package com.fbaron.ims.inventory.model;

public enum MovementType {
    INBOUND,   // Stock addition
    OUTBOUND   // Stock removal
}
```

### Type Details

<Tabs>
  <Tab title="INBOUND">
    **Stock Addition**
    
    Represents movements that increase inventory stock:
    - Purchase orders from suppliers
    - Returns from customers
    - Production output
    - Initial stock setup
    - Stock corrections (increases)
    
    **Impact:** Adds to total stock balance
  </Tab>
  
  <Tab title="OUTBOUND">
    **Stock Removal**
    
    Represents movements that decrease inventory stock:
    - Customer sales
    - Damaged/defective items
    - Returns to suppliers
    - Internal consumption
    - Stock corrections (decreases)
    
    **Impact:** Subtracts from total stock balance
    
    **Validation:** System checks sufficient stock exists before allowing outbound movements
  </Tab>
</Tabs>

## Design Patterns

### Event Sourcing Approach

Inventory movements follow an event-sourcing pattern where:

1. **Immutable Events:** Each movement is an immutable event that is never modified or deleted
2. **Current State Derived:** The current stock balance is calculated by aggregating all movements
3. **Complete History:** Full audit trail is maintained automatically

```java
// Stock calculation formula
Integer currentStock = sumOfAllInboundQuantities - sumOfAllOutboundQuantities;
```

<Expandable title="Benefits of Event Sourcing">
  - **Complete Audit Trail:** Every stock change is recorded with timestamp and reason
  - **Historical Analysis:** Can reconstruct stock levels at any point in time
  - **No Data Loss:** Past movements are never modified or deleted
  - **Debugging:** Easy to trace why stock is at current level
  - **Compliance:** Meets audit requirements for inventory tracking
</Expandable>

### Builder Pattern

Movements are constructed using the Builder pattern:

```java
InventoryMovement movement = InventoryMovement.builder()
    .product(product)
    .quantity(50)
    .type(MovementType.INBOUND)
    .reason("Purchase order #PO-2026-001")
    .createdAt(LocalDateTime.now())
    .build();
```

## Usage Examples

### Creating an Inbound Movement

```java
// Business logic in RegisterMovementUseCase.inbound()
Product product = productRepository.findById(productId)
    .orElseThrow(() -> new ProductNotFoundException(productId));

InventoryMovement movement = InventoryMovement.builder()
    .product(product)
    .quantity(100)
    .type(MovementType.INBOUND)
    .reason("Initial stock from supplier")
    .createdAt(LocalDateTime.now())
    .build();

return movementRepository.save(movement);
```

### Creating an Outbound Movement with Validation

```java
// Business logic in RegisterMovementUseCase.outbound()
Product product = productRepository.findById(productId)
    .orElseThrow(() -> new ProductNotFoundException(productId));

// Validate sufficient stock
Integer currentBalance = calculateStock(productId);
if (currentBalance < quantity) {
    throw new InsufficientStockException(
        "Cannot exit " + quantity + " units. Current balance is only " + currentBalance
    );
}

InventoryMovement movement = InventoryMovement.builder()
    .product(product)
    .quantity(quantity)
    .type(MovementType.OUTBOUND)
    .reason(reason)
    .createdAt(LocalDateTime.now())
    .build();

return movementRepository.save(movement);
```

### Calculating Stock Balance

```java
public Integer calculateStock(UUID productId) {
    productRepository.findById(productId)
        .orElseThrow(() -> new ProductNotFoundException(productId));

    Integer inputs = movementRepository.findTotalInputs(productId);
    Integer outputs = movementRepository.findTotalOutputs(productId);

    return inputs - outputs;
}
```

## Validation Rules

<Warning>
  Multiple levels of validation ensure data integrity:
  1. **DTO Validation** - Jakarta Bean Validation at REST layer
  2. **Business Logic Validation** - Domain rules in use cases
  3. **Stock Validation** - Prevents negative stock on outbound movements
</Warning>

| Field | Validation | Enforced At | Error Message |
|-------|------------|-------------|---------------|
| productId | `@NotNull`, exists | DTO + Business | "Product id is required" / "Product not found" |
| quantity | `@Positive`, > 0 | DTO + Business | "Quantity must be greater than zero" |
| reason | `@NotBlank` | DTO | "Reason is required" |
| stock (outbound) | balance >= quantity | Business | "Cannot exit X units. Current balance is only Y" |

## Exception Handling

Domain exceptions specific to inventory movements:

### InsufficientStockException

```java
public class InsufficientStockException extends RuntimeException {
    public InsufficientStockException(String message) {
        super(message);
    }
}
```

**Thrown when:** Attempting to create an outbound movement that would result in negative stock

**HTTP Status:** 400 Bad Request

**Example:** `"Cannot exit 50 units. Current balance is only 20"`

### InvalidMovementQuantityException

```java
public class InvalidMovementQuantityException extends RuntimeException {
    public InvalidMovementQuantityException(Integer quantity) {
        super("Movement quantity must be greater than zero. Received: " + quantity);
    }
}
```

**Thrown when:** Quantity is null or not positive

**HTTP Status:** 400 Bad Request

### ProductNotFoundException

**Thrown when:** Attempting to create a movement for a non-existent product

**HTTP Status:** 404 Not Found

## Repository Operations

```java
// Query operations
public interface InventoryMovementQueryRepository {
    Integer findTotalInputs(UUID productId);
    Integer findTotalOutputs(UUID productId);
}

// Command operations
public interface InventoryMovementCommandRepository {
    InventoryMovement save(InventoryMovement movement);
}
```

<Expandable title="Repository Implementation Notes">
  - `findTotalInputs()` - Sums quantity where type = INBOUND
  - `findTotalOutputs()` - Sums quantity where type = OUTBOUND
  - `save()` - Persists movement as immutable event
  - No update or delete operations (event sourcing principle)
</Expandable>
